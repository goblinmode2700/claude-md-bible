Sample Pattern

Session modes prevent mode confusion

The single most impactful pattern. AI mixes feature work with refactoring with testing, producing muddled commits. Session modes fix this.

CLAUDE.md
## Session Modes

# Every session operates in exactly one mode.
# If no mode is stated, ask before proceeding.

| Mode | Tag | Purpose |
|---|---|---|
| [WRITE] | Implement features per the current story |
| [REFACTOR] | Improve structure — no new features |
| [TEST] | Write or fix tests only — no feature code |
| [DOCUMENT] | Update docs, specs, ADRs — no code |

Rule: After every 2-3 [WRITE] sessions,
a [REFACTOR] session is mandatory.
